Change Preventer

Definition

Change Preventers Code Smell mengacu pada praktik atau pola pengkodean yang menyulitkan atau rumit untuk memodifikasi atau memperluas basis kode. Biasanya muncul karena keterkaitan antar komponen, kurangnya abstraksi, atau pilihan desain yang kaku. Akibatnya, melakukan perubahan pada kode, seperti menambahkan fitur baru atau mengakomodasi persyaratan yang terus berkembang, menjadi tantangan dan rawan kesalahan.

Example

Jika dilihat secara sekilas, class diatas tidak memiliki kesalahan. Namun misalkan diperlukan fungsionalitas tambahan seperti meng-export file ke dalam format yang berbeda, banyak perubahan perlu dilakukan. Oleh sebab itu, class diatas perlu diganti dan method yang perlu dikeluarkan dari class tersebut.

Treatment

Cara memperbaikinya adalah dengan membuat class melakukan apa yang class itu perlu lakukan. Jika ada method yang tidak sesuai dengan class, maka method tersebut perlu dikeluarkan dan dimasukkan ke class lainnya. Pada gambar diatas method "ExportTreatmentHistory" dipisah dari class Patient karena tidak memiliki kolerasi dengan class tersebut dan dimasukkan ke dalam class terpisah.

Dispensable

Definition

Dispensable Code Smell mengacu pada bagian kode yang tidak diperlukan atau berlebihan, dan tidak memiliki tujuan fungsional dalam perangkat lunak. Bagian kode ini mungkin merupakan sisa dari iterasi sebelumnya, diperkenalkan sebagai perbaikan sementara, atau hanya ditulis tanpa pertimbangan yang matang. Kode yang dapat dibuang dapat meningkatkan kompleksitas, menurunkan keterbacaan, dan berpotensi menimbulkan bug atau masalah pemeliharaan.

Example

Pada kasus diatas, nama function tidak memberi penjelasan akan apa yang akan dilakukan function tersebut. Oleh karena itu ditambahkanlah comment diatas function untuk menjelaskan apa yang dilakukan function tersebut. Namun menulis code seperti ini bukanlah hal yang baik karena sebenarnya comment tersebut tidak diperlukan.

Treatment

Untuk memperbaiki code tersebut hal yang dapat dilakukan adalah dengan mengubah nama function tersebut sesuai dengan yang dilakukan function tersebut. Dengan begitu comment yang berada diatas function dapat dihapus karena ketika membaca nama function, orang yang membaca akan langsung memahami apa yang dilakukan function tersebut.
